Cloud Service Models — IaaS vs PaaS vs SaaS

ModelWhat Microsoft ManagesWhat You ManageExample
IaaS (Infrastructure as a Service)Physical hardware, networking, virtualisationOS, runtime, middleware, apps, dataAzure Virtual Machines, Azure VNet
PaaS (Platform as a Service)Everything in IaaS + OS + runtime + middlewareApplications and data onlyAzure App Service, Azure SQL Database, Azure Functions
SaaS (Software as a Service)Everything — the full applicationYour data and user configuration onlyMicrosoft 365, Dynamics 365, Salesforce
⭐ Memory Trick

IaaS = most control, most responsibility. PaaS = you manage app + data only. SaaS = just use it, Microsoft handles everything else.

🌐

Cloud Deployment Models

Public Cloud

  • Resources owned & operated by Microsoft
  • Shared infrastructure (multi-tenant)
  • Access over internet
  • Pay-as-you-go pricing
  • Example: Azure, AWS, GCP

Private Cloud

  • Dedicated to one organisation
  • Hosted on-premises or by a provider
  • Full control & customisation
  • Higher cost & responsibility
  • Example: Azure Stack, on-prem DC

Hybrid Cloud

  • Combination of public + private
  • Data can move between environments
  • Greater flexibility
  • Can keep sensitive data on-prem
  • Example: Azure Arc, Azure Stack HCI
⚡

Key Cloud Benefits — The 6 Advantages

High Availability
Services remain accessible even during failures — guaranteed by SLA (e.g. 99.9%)
Scalability
Increase resources to handle growing demand (vertical = bigger VM; horizontal = more VMs)
Elasticity
Automatically scale up and down based on actual demand in real time
Agility
Deploy and configure cloud resources quickly — minutes vs months for on-prem
Geo-Distribution
Deploy apps and data to datacentres worldwide, close to users, low latency
Disaster Recovery
Replicate data across regions; failover to another region if primary fails
💰

CapEx vs OpEx — Cloud Economics

CapEx (Capital Expenditure)

  • Upfront investment in physical infrastructure
  • High initial cost, long depreciation cycle
  • On-premises datacentres
  • You own and manage the hardware
  • Fixed capacity — can under/over-provision

OpEx (Operational Expenditure)

  • Pay-as-you-go — use what you need
  • No upfront costs; ongoing subscription/usage fees
  • Cloud computing model
  • Microsoft owns and manages hardware
  • Scale up/down anytime based on demand
⭐ Exam Tip

Cloud = OpEx model. Benefits: deduct as business expense, no guessing future capacity, only pay for what you use.

⭐ Points to Remember — Cloud Concepts

  • Consumption-based model = you pay only for resources you use (OpEx), no more guessing capacity
  • Scalability = planned growth; Elasticity = automatic real-time scaling
  • Fault tolerance = system continues even if a component fails
  • Shared responsibility model: Microsoft secures physical, you secure your data and access
  • High availability measured by SLA — 99.9% = ~8.7 hrs downtime/year; 99.99% = ~52 mins/year

Azure Global Infrastructure

ConceptDefinition
RegionGeographic area containing one or more datacentres (e.g. UK South, East US). Choose region for latency, compliance, and service availability.
Availability ZonePhysically separate datacentre within a region with independent power, cooling, networking. Protects against datacentre failure. Minimum 3 zones per region.
Region PairTwo regions paired together for disaster recovery. Updates rolled out sequentially, not simultaneously. Data residency maintained.
GeographyDiscrete market (e.g. Europe, Americas) that contains two or more regions. Preserves data residency and compliance.
Sovereign RegionsIsolated instances of Azure (e.g. Azure Government, Azure China) for regulatory compliance.

Core Azure Storage Services

ServiceDescriptionUse Case
Blob StorageObject storage for unstructured data — files, images, videos, backupsData lake, media files, backups, static websites
Azure FilesFully managed file shares (SMB protocol) accessible from cloud and on-premReplace on-prem file servers, shared configuration files
Azure Queue StorageMessage queuing for async communication between app componentsDecouple app components, background job processing
Azure Table StorageNoSQL key-attribute store for semi-structured dataWeb app data, address books, metadata
Azure Disk StorageManaged disks for Azure VMs (HDD, SSD, Ultra Disk)VM operating system and data disks

Blob Storage Access Tiers

  • Hot — frequently accessed; lowest access cost, highest storage cost
  • Cool — infrequently accessed (30+ days); lower storage cost
  • Cold — rarely accessed (90+ days); even lower storage cost
  • Archive — long-term retention (180+ days); lowest storage cost, high retrieval cost/time

Storage Redundancy Options

  • LRS — 3 copies in one datacentre (cheapest)
  • ZRS — 3 copies across 3 Availability Zones in one region
  • GRS — LRS + async replication to paired region
  • GZRS — ZRS + async replication to paired region (most resilient)
🌐

Core Azure Networking Services

Azure Virtual Network (VNet)
Private network in Azure — isolates and segments resources; subnets for organisation
VNet Peering
Connect two VNets privately — traffic stays on Microsoft backbone, not internet
Azure VPN Gateway
Encrypted connection between Azure VNet and on-premises over public internet (IPsec)
Azure ExpressRoute
Private, dedicated connection from on-premises to Azure — NOT over internet; higher reliability & speed
Azure DNS
Host DNS domains in Azure for name resolution using Microsoft infrastructure
Azure Load Balancer
Distributes traffic across VMs at Layer 4 (TCP/UDP) for high availability
Azure Application Gateway
Layer 7 load balancer with WAF (Web Application Firewall) for HTTP/HTTPS apps
Azure CDN
Content Delivery Network — caches static content at edge locations worldwide for low latency
⭐ Exam Tip — VPN vs ExpressRoute

VPN Gateway = encrypted over internet (lower cost, lower speed). ExpressRoute = private dedicated circuit (higher cost, higher reliability, not over internet). ExpressRoute does NOT traverse the public internet.

⭐ Points to Remember — Azure Services

  • VMs = IaaS (you manage OS); App Service = PaaS (Microsoft manages OS); Functions = Serverless
  • Availability Zones = protect against datacentre failure; Region Pairs = protect against region failure
  • Archive tier = cheapest storage but takes hours to rehydrate — not for active data
  • ExpressRoute = private circuit, no public internet; VPN Gateway = encrypted but over internet
  • Load Balancer = Layer 4 (TCP/UDP); Application Gateway = Layer 7 (HTTP/HTTPS) with WAF

Identity & Access Management

Microsoft Entra ID (Azure AD)
Cloud-based identity and access management service — single sign-on, MFA, Conditional Access
Multi-Factor Authentication (MFA)
Requires 2+ forms of verification: something you know + something you have + something you are
Conditional Access
Enforce access policies based on signals: user, location, device compliance, app, risk level
Role-Based Access Control (RBAC)
Assign roles to users/groups at subscription, resource group, or resource scope (Owner, Contributor, Reader)
Single Sign-On (SSO)
Authenticate once, access multiple apps — less password fatigue, centralised access control
Zero Trust Model
Verify explicitly, Use least privilege access, Assume breach — never trust, always verify
🏛️

Azure Governance Tools

ToolPurpose
Azure PolicyDefine and enforce organisational standards — prevent non-compliant resources from being created (e.g. require tags, restrict VM sizes, enforce regions)
Azure BlueprintsDeploy a complete environment (policies, role assignments, resource groups, ARM templates) as a repeatable package
Management GroupsOrganise subscriptions into a hierarchy for governance — apply policies and RBAC at scale above subscription level
Resource TagsMetadata key-value pairs applied to resources for cost tracking, automation, and management (NOT inherited by default)
Resource LocksPrevent accidental deletion or modification — ReadOnly (no changes) or Delete (can't delete, can modify)
Azure AdvisorPersonalised best-practice recommendations across: Reliability, Security, Performance, Cost, Operational Excellence

⭐ Points to Remember — Security & Governance

  • Azure Policy = ENFORCE rules; Azure Advisor = RECOMMEND improvements (doesn't enforce)
  • Resource Lock hierarchy: Lock at resource group → applies to all resources within
  • RBAC scope: Management Group → Subscription → Resource Group → Resource (inherits downward)
  • Microsoft Defender for Cloud = Security posture management; Microsoft Sentinel = SIEM/SOAR (threat detection)
  • DDoS Basic = free/automatic; DDoS Standard = paid, enhanced protection with dedicated support
  • Tags are NOT inherited by child resources — must be applied explicitly or via Policy

Azure Cost Management Tools

Azure Pricing Calculator
Estimate monthly cost BEFORE deploying — configure services and see price estimates
Total Cost of Ownership (TCO) Calculator
Compare on-premises costs vs Azure costs — helps build business case for cloud migration
Azure Cost Management + Billing
Monitor, analyse, and optimise actual spending after deployment; set budgets and alerts
Azure Reservations
Pre-pay for 1 or 3 years — save up to 72% vs pay-as-you-go for predictable workloads
📋

Factors That Affect Azure Cost

  • Resource type — different services have different pricing models (per-second, per-GB, per-request)
  • Region — prices vary by Azure region; some regions are cheaper than others
  • Bandwidth — ingress (data IN to Azure) is generally free; egress (data OUT) is charged
  • Subscription type — Free, Pay-as-you-go, Enterprise Agreement, CSP (partner) all have different rates
  • Azure Marketplace — third-party software may have additional licensing costs
  • Azure Hybrid Benefit — use existing Windows Server / SQL Server licences in Azure to save money
  • Spot VMs — up to 90% discount; workload can be interrupted if Azure needs capacity back

⭐ Composite SLA

When services depend on each other, multiply SLAs: VM (99.9%) × SQL Database (99.99%) = 99.89% composite SLA. More dependencies = lower composite SLA.

Common Exam Traps — Don't Fall For These

❌ TRAP: "Availability Sets and Availability Zones are the same thing"
✅ CORRECT: Availability Sets = protect against hardware failures within ONE datacentre. Availability Zones = physically separate datacentres within a region. Zones = better protection.
❌ TRAP: "ExpressRoute goes over the internet but with encryption"
✅ CORRECT: ExpressRoute is a PRIVATE dedicated circuit — it does NOT traverse the public internet at all. VPN Gateway uses the internet with IPsec encryption.
❌ TRAP: "Tags are automatically inherited by child resources"
✅ CORRECT: Tags are NOT inherited. A tag on a Resource Group does not automatically apply to resources inside. Use Azure Policy to enforce tag inheritance.
❌ TRAP: "Azure Policy prevents all policy violations"
✅ CORRECT: Azure Policy can Deny, Audit, or Append — but existing non-compliant resources are not automatically remediated. You must run remediation tasks for existing resources.
❌ TRAP: "Azure Advisor enforces best practices"
✅ CORRECT: Azure Advisor only RECOMMENDS — it doesn't enforce anything or make any changes. It's advisory only.
❌ TRAP: "A ReadOnly lock prevents resource deletion"
✅ CORRECT: ReadOnly prevents BOTH modification AND deletion. Delete lock only prevents deletion but allows modification.
❌ TRAP: "Composite SLA of two services is the average of both SLAs"
✅ CORRECT: Composite SLA = multiply the individual SLAs (e.g. 99.9% × 99.9% = 99.8%), not average them.
❌ TRAP: "Spot VMs are always running"
✅ CORRECT: Spot VMs CAN BE EVICTED by Azure with 30 seconds notice when Azure needs capacity. Only use for interruptible, fault-tolerant workloads.
