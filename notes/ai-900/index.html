AI vs ML vs Deep Learning — The Hierarchy

AI (Broad field)
⊃
Machine Learning (Subset of AI)
⊃
Deep Learning (Subset of ML)
⊃
Gen AI (Subset of DL)

AI encompasses all systems that mimic human intelligence. ML learns from data. Deep learning uses neural networks with many layers. Generative AI uses foundation models (LLMs) to generate content.

⭐ Points to Remember — AI Concepts

  • AI = computers that mimic human decision-making
  • Machine Learning = teaches computers by providing examples (training data)
  • Deep Learning = multi-layer neural networks, basis of modern AI
  • Generative AI = AI that generates NEW content (text, images, code, audio)
  • Azure AI Services = pre-built APIs; Azure ML = build-your-own models

Domain 2 — Machine Learning on Azure

25–30%

Types of Machine Learning

TypeHow It WorksExample Use Case
Supervised LearningLearns from labelled training data (input + correct output)Email spam detection, house price prediction
Unsupervised LearningFinds patterns in unlabelled dataCustomer segmentation, anomaly detection
Reinforcement LearningAgent learns by trial and error, rewarded for correct actionsGame playing AI, robot navigation

Regression

  • Predicts a numeric value
  • Example: "How much will this house cost?"
  • Output: continuous number (£350,000)

Classification

  • Predicts a category/class
  • Example: "Is this email spam or not?"
  • Binary (2 classes) or multi-class (many)

Clustering

  • Unsupervised — groups similar items
  • Example: "Group these customers by behaviour"
  • No labels needed — patterns emerge

Anomaly Detection

  • Identifies unusual patterns
  • Example: "Flag this unusual bank transaction"
  • Used for fraud detection, quality control

Model Evaluation Metrics — Know the Difference

MetricUsed ForDefinition
Mean Absolute Error (MAE)RegressionAverage absolute difference between predicted and actual values
Root Mean Square Error (RMSE)RegressionSquare root of average squared errors — penalises large errors more
R² (Coefficient of Determination)RegressionHow well model explains variance; 1.0 = perfect, 0 = no better than mean
AccuracyClassification% of correct predictions out of all predictions
PrecisionClassificationOf predicted positives, how many are actually positive
Recall (Sensitivity)ClassificationOf actual positives, how many did we correctly identify
F1 ScoreClassificationHarmonic mean of Precision and Recall — balance metric
AUC-ROCClassificationArea under ROC curve; 1.0 = perfect, 0.5 = random guessing
⭐ Exam Tip — Regression vs Classification Metrics
  • Regression = numeric prediction → use MAE, RMSE, R²
  • Classification = category prediction → use Accuracy, Precision, Recall, F1, AUC
  • If asked "which model is better?" for classification → higher F1/AUC is generally better

⭐ Points to Remember — Azure ML

  • AutoML tries many algorithms automatically — saves time, no ML expertise needed
  • Designer = drag-and-drop, no code needed; great for experimenting
  • Model training = Compute Cluster; Real-time inference = Compute Instance or managed endpoint
  • Overfitting = model memorised training data, fails on new data; fix with more data or regularisation
  • Feature engineering = selecting/transforming input variables to improve model performance

Azure AI Document Intelligence (Form Recognizer)

Extracts structured data from forms, receipts, invoices, ID cards and documents. Uses prebuilt models for common document types or custom models for specific layouts.

Prebuilt Models

  • Receipt → total, items, merchant, date
  • Invoice → vendor, line items, amounts
  • ID Document → name, DOB, ID number
  • Business Card → name, company, phone

Custom Models

  • Train on your own documents
  • Label fields manually in AI Studio
  • Need at least 5 sample documents
  • Good for unique form layouts
🔎

Azure AI Search (Knowledge Mining)

Extracts and indexes insights from large volumes of unstructured content (PDFs, images, audio) using an enrichment pipeline powered by Azure AI Skills.

Data Source (Blob, SQL)
→
Indexer (crawls data)
→
Skillset (AI enrichment)

⭐ Points to Remember — Computer Vision

  • OCR = extracting text from images (read tool in Azure AI Vision)
  • Object Detection adds bounding box coordinates; Image Classification does not
  • Custom Vision = Azure AI Vision service for training custom image classifiers and object detectors
  • Azure AI Document Intelligence = formerly Form Recognizer (same service, renamed)
  • Azure AI Search = indexed search + AI enrichment pipeline (skills)

Azure AI Bot Service & Language Understanding

Azure AI Bot Service

  • Framework for building conversational bots
  • Integrates with Teams, WhatsApp, web chat, etc.
  • Uses CLU or Question Answering for language understanding
  • Power Virtual Agents = low-code bot builder (now Microsoft Copilot Studio)

Conversational Language Understanding (CLU)

  • Understands user intent and entities in natural language
  • Train with example utterances per intent
  • Example: "Book a flight to London" → Intent=BookFlight, Entity=London
  • Successor to LUIS (Language Understanding Intelligent Service)
🔊

Azure AI Speech Services

Speech-to-Text (STT)
Converts spoken audio to written text — transcription, live captions
Text-to-Speech (TTS)
Converts written text into spoken audio — natural-sounding neural voices
Speech Translation
Translates spoken language in real-time across 90+ languages
Speaker Recognition
Identifies who is speaking based on voice biometrics

⭐ Points to Remember — NLP

  • Azure AI Translator = text translation; Azure AI Speech Translation = spoken language translation
  • Question Answering project = create FAQ from docs; no ML training needed for extraction mode
  • CLU replaces LUIS — same concepts (intents + entities) but newer service name
  • Sentiment Analysis returns a score AND a label (Positive/Negative/Neutral/Mixed)
  • NER identifies real-world entities: Person, Organisation, Location, DateTime, Quantity

Large Language Models (LLMs) — Core Concepts

LLM
Large Language Model — trained on massive text data to predict and generate text
Foundation Model
Pretrained model (GPT-4, etc.) used as base for many downstream tasks
Token
Smallest unit of text an LLM processes; roughly ¾ of a word on average
Context Window
Max tokens the model can consider in one interaction (input + output)
Prompt
The input text/instruction sent to the model
Completion
The model's generated response to a prompt
⚙️

Azure OpenAI Service

Azure OpenAI Service provides REST API access to OpenAI's powerful models (GPT-4, DALL-E, Whisper, Embeddings) with Azure's enterprise security, compliance, and regional deployment.

Models Available

  • GPT-4 / GPT-3.5 = text generation & chat
  • DALL-E = image generation from text
  • Whisper = speech-to-text
  • Embeddings = semantic search & similarity

Azure vs OpenAI.com

  • Same models, different infrastructure
  • Azure = private deployment in your region
  • Data stays within Azure — NOT used for training
  • Enterprise compliance (SOC 2, HIPAA, etc.)

Prompt Engineering — Key Techniques

TechniqueDescriptionWhen to Use
Zero-ShotNo examples — just the task descriptionSimple tasks, general instructions
Few-ShotInclude 2–5 examples in the promptSpecific format, tone, or pattern needed
Chain-of-Thought (CoT)Ask model to "think step by step"Math problems, reasoning, multi-step logic
System PromptInstructions that define model's behaviour/personaChatbots, consistent tone, constraints
⭐ Exam Tip

Grounding = providing specific facts/documents in the prompt so the model answers based on real data, not general training (this is the basis of RAG — Retrieval Augmented Generation)

Responsible AI — Microsoft's 6 Principles

Always on exam
🎯

The 6 Principles of Responsible AI (Memorise These)

PrincipleWhat it MeansExample
FairnessAI should treat all people fairly, without bias or discriminationLoan approval model tested across demographic groups
Reliability & SafetyAI should work reliably and safely; fail gracefullyAutonomous vehicle tested extensively before deployment
Privacy & SecurityAI should protect personal data and resist attacksModel trained on anonymised data; secure APIs
InclusivenessAI should empower everyone, including people with disabilitiesScreen reader support, multi-language, accessibility features
TransparencyAI systems should be explainable and understandableExplaining why a loan was declined by the model
AccountabilityPeople should be responsible for AI systems and their outcomesHuman review required before AI-generated medical diagnosis

⭐ Responsible AI — Never Forget

  • All 6 principles: Fairness · Reliability & Safety · Privacy & Security · Inclusiveness · Transparency · Accountability
  • Transparency ≠ making source code public — it means making AI decisions explainable
  • Accountability = humans remain responsible for AI outputs, even when AI makes decisions
  • Microsoft AI Principles apply to Azure OpenAI Service — responsible use policies enforced
  • Content filters in Azure OpenAI block harmful content (hate speech, violence, self-harm, sexual)

Common Exam Traps — Don't Fall For These

❌ TRAP: "Image Classification and Object Detection are the same"
✅ CORRECT: Classification = whole image gets a label. Object Detection = individual objects identified WITH their bounding box location.
❌ TRAP: "Regression predicts a category, Classification predicts a number"
✅ CORRECT: OPPOSITE. Regression = numeric output (price, temperature). Classification = category (spam/not spam, dog/cat).
❌ TRAP: "Azure OpenAI uses your data to retrain its models"
✅ CORRECT: Microsoft guarantees your data is NOT used for training. Your prompts stay private within your Azure subscription.
❌ TRAP: "LUIS is still the recommended NLU service"
✅ CORRECT: LUIS is deprecated. The current service is CLU (Conversational Language Understanding) within Azure AI Language.
❌ TRAP: "Transparency means publishing the AI source code"
✅ CORRECT: Transparency = making AI decisions understandable and explainable to users. It does NOT mean open-sourcing everything.
❌ TRAP: "Unsupervised learning uses labelled training data"
✅ CORRECT: Unsupervised = NO labels. The algorithm finds patterns on its own. Supervised = labelled data with correct answers.
❌ TRAP: "Azure AI Search is just a search engine like Google"
✅ CORRECT: Azure AI Search includes an AI enrichment pipeline (skills) that can extract entities, translate, and index insights from unstructured data.
❌ TRAP: "Form Recognizer and Azure AI Document Intelligence are different services"
✅ CORRECT: They are the SAME service — Form Recognizer was renamed to Azure AI Document Intelligence.
